# Web Application Structure

This document outlines the structure and architecture of the Flashcard Web Application that interfaces with your n8n flashcard workflow.

## Project Overview

A modern React.js web application designed to create, manage, and study interactive flashcards generated by your n8n AI workflow. The application provides a complete study experience with multiple modes, real-time progress tracking, and export capabilities.

## Core Components

### 1. UploadArea
File: `components/Features/UploadArea.tsx`

The main interface for uploading study materials. Features:
- Drag & drop support for PDF, TXT, DOCX files
- Direct text input area
- Real-time preview of uploaded files
- File type validation and error handling
- Progress indicators during upload

### 2. Flashcard
File: `components/Features/Flashcard.tsx`

Interactive flashcard component with:
- Dual-sided design (question/answer)
- Difficulty indicators (easy/medium/hard)
- Concept categorization
- Smooth flip animations
- Responsive design for mobile and desktop

### 3. StudyPage
File: `pages/StudyPage.tsx`

Main study interface featuring:
- Interactive card navigation
- Study mode selector (basic/timed/spaced-repetition)
- Progress tracking
- Answer evaluation system
- Time tracking and scoring

### 4. API Service
File: `services/apiService.ts`

Integration layer for n8n workflow:
- File upload and processing
- Session management
- Flashcard retrieval
- Export functionality (CSV/JSON)
- Error handling and retries

## Project Notes

### Dependencies
- Core React ecosystem with Vite for rapid development
- Tailwind CSS for utility-based styling
- Lucide React for clean, consistent icons
- Framer Motion for smooth user interactions

### File Naming Conventions
- **Components**: PascalCase (e.g., `UploadArea.tsx`)
- **Functions**: camelCase (e.g., `handleUpload()`)
- **Variables**: camelCase (e.g., `isUploading`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `MAX_FILE_SIZE`)

### Code Standards
- **Comments**: English, Allman-style braces for JS Code nodes
- **TypeScript**: strict mode enabled
- **ESLint**: Custom configuration for React projects
- **Prettier**: Code formatting consistency

This structure provides a solid foundation for your flashcard web application with clear separation of concerns, maintainable code, and a path for future enhancements.
